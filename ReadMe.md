### Структура файлов
Были реализованы два варианта атомарных 64-х битных счетчика. Основная сложность - согласовать работу n-поток при обработке полей счетчика. 

1. _Part 1_: В *Части 1* у 64-х битного счетчика есть всего два 32-х битных поля. Реализация расположена в директории `Part_1/`. Класс счетчика, поля и методы объявлены в `Part_1/workspace/include/counter.hpp`, реализация вынесена в `Part_1/workspace/include/counter.cpp`

2. _Part 2_: В *Части 2* у 64-х битного счетчика может быть несколько `8*2^n (0 <= n <= 2)` битных полей. Реализация расположена в директории `Part_2/`. Класс счетчика, поля и методы объявлены и реализованы в `Part_2/workspace/include/counter.hpp`. Реализация выполнена прямо в `.hpp` в связи с особенностями работы с `template<typename ...>`

3. Проект использует google-тесты, которые подключаются при сборке проекта.

### Установка, сборка и запсук

```Bash
git clone https://github.com/7RosenRot/NLRC_Parallel_Runtimes.git
cd NLRC_Parallel_Runtimes/Part_<1/2>/

cmake -S . -B build
cmake --build build --parallel --config=Release

build/tests/Release/Part_<1/2>Tests.exe
```

*! Обратите внимание, что проекты находятся в разных директориях, для перехода и запуска используйте выбор <1/2> !*
