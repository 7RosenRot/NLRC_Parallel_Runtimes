### Постановка задачи
Теперь, предположим, нам пришлось разбить наш счетчик на несколько разных по размеру целочисленных счетчиков. (В моем случае - `uint16_t + uint16_t + uint32_t`). Ключевое отличие - мы не можем использовать разбиение на некоторое количество одинаковых полей. В итоге у нас есть `uint64_t` контейнер, в который будут последовательно складываться поля счетчика.

### Универсация для n-элементов
Чтобы использовать последовательность счетчиков, будем использовать картеж `std::tuple<std::atomic<Args>&...>`, что позволит нам обращаться к ним по индексу.
1. Так как в итоге нам придется собирать несколько счетчиков в один, реализуем механизм "упаковки". 

*Псевдокод*
```C++
template <typename T, typename... Other>
uint64_t recursive_read(shift, T& value, Other& next_value) const {
  uint64_t val = value;
  
  uint64_t result = (value << shift) | recursive_read(shift + sizeof(value_type) * 8, next_value);
  
  return result;
}
```

Фактически - просто сдвиг на `shift + sizeof(value_type) * 8`. То есть, если у нас есть 3 счечика - `uint16_t + uint16_t + uint32_t`, упаковка будет выглядеть следующим образом:

```C++
1.  0 ...  0 <- uint16_t
[63]^   [0]^

2.  0 ... 1 ... 1 <- uint16_t
[63]^ [15]^  [0]^

3.  1 ... 1 ... 1 ... 1 <- uint32_t
[63]^ [31]^ [15]^  [0]^
```

2. Так как рано или поздно счетчик переполнится, необходимо предусмотреть метод переноса разряда. Если при увеличении, предыдущее значение уже достигло максимуму - *обнаружено переполнение* -> переходим к следующему шагу рекурсии, передавая в нее следующие по очереди элементы.

*Псевдокод*
```C++
template <typename T, typename... Other>
void recursive_propagate(T& current, Other&... other) {
  T prev = current;
  current += 1;
  
  if (prev == max()) {
    recursive_propagate(other...);
  }
}
```

3. Для того, чтобы выполнить сброс итоговоого счетчика, необходимо обнулить *каждый из n-полей*. Для этого предусмотрим метод, обратный чтению. То есть теперь мы идем по счетчику в обратом порядке и выполняем сдвиг на количество бит одного поля. 

*Псевдокод*
```C++
template <typename T, typename... Other>
void recursive_reset(value, shift, T& current, Other&... other) {
  current = value >> shift;
  
  recursive_reset(value, shift + sizeof(T) * 8, other...);
}
```

### Увелиение целевого счетчика и возврат предыдущего значени
Метод `fetch_add()` для данной модификации алгоритма во многом схож с его реализацией в *Части 1*.
1. Получение текущего значения счетчика. Мы также два раза считываем данные и возвращаем из только в том слчае, когда считанное значение соответсвует повторному чтению.
2. Затем мы увеличивает значение самого младшего из полей, :

*Псевдокод*
```C++
auto& low_atomic = parts_[0];

if (low_atomic.compare_exchange_weak(crt_low, upd_low)) {
  if (upd_low == 0) {
    std::apply(recursive_propagate, parts_);
  }
  
  return crt_value;
}
```

3. Метод рекурсивного инкремена применится для младшего поля только в случе успешного CAS и зафиксированного переполнения `upd_low == 0`.

### Объяснение используемых моделй памяти.
1. Для *чтения сегмента* используется флаг `std::memory_order_acquire` - таким образом гарантируется, что наш поток видит все последние изменения
2. Для *модификации* используем флаг `std::memory_order_acq_rel` - таким образом гарантируем, что чтение выполняется немедленно, а публикация данных - незамедлительно
3. При *увеличении/сбросе* счетчика используем флаг `std::memory_order_release` - таким образом грантируем, что все читающие потоки получат актуальную информацию (см. пункт "Для *чтения сегмента*")
